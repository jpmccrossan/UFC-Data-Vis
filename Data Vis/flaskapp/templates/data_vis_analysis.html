CMT218 Data Analysis and Visualisation Creation

Data Visualisation Analysis and Discussion



My message is: “Helping UFC fans find insight and meaning by comparing and contrasting data from the best UFC fighters for deeper engagement with the sport.”



Data Fighter V1 design process: the ‘why’
As an MMA enthusiast and practitioner, I set out to create a UFC fighter statistics data visualisation project for fans and satisfy their data needs (Iliinsky, N. and Steele, J. (2011)). I sourced my dataset from Kaggle (Aggarwal, K. (2018)) for its up-to-date, mostly clean, and well-organized data spanning 1996-2024. Cleaning the headers for better readability and intuitiveness was crucial (Chai, C.P. (2020)), although the extensive dataset also posed challenges of either oversimplification or overwhelming detail with 2500 rows.

Recognizing that the majority of UFC viewers focus on the top male fighters based on PPV information (Davies, B. (2023)), I filtered the data accordingly, acknowledging the sexist bias this introduced and aiming to rectify it in Version 2.

I formulated three assumptions for guiding my design choices and inferred from them to help me tell a story (Duarte, N. (2019)). This is the ‘why’ behind my data dashboard:

Assumption 1:
I looked at the data and decided having ways to group and categorise would help the TA find the data visualisations more meaningful and insightful. This is because it helps align the question (my message), data (by narrowing the focus of what data is ‘speaking’) and visualisation (by not cluttering the charts and enhancing clarity) (Kaiser, F. (2014)). This meant rendering the data dynamically would be best.

Inferred design choice: grouping and categorizing data for user insight
I chose a web app data dashboard to increase user interactivity and useability, with data filtering and rendering capabilities.
To assist user insight, I used fighter weight class based data grouping to make the data more interpretable: offensive data (when a fighter was attacking), defensive data (when a fighter was being attacked or defending) and body data (stats about the physical make-up of the fighter). A data design choice I made was that as weight is generally consistent across a weight class this was of lesser importance to the TA.

I grouped headers into these thematic areas on the weight class and data fighter pages to visually link them and assist users page navigation and data interpretation (Muth, L.C. (2020)).

Assumption 2:
UFC fans enjoy the fast-paced dopamine rich experiences of fights so they would most likely want to be able to quickly gain insight from the data with minimal effort (Cools, R. (2019)).

Inferred design choice: prioritizing quick and effortless data interpretation
These assumptions led to me designing a web app dashboard with filtering capabilities and a minimalist layout for enhanced user experience using iterative wireframe development and implementation (Gould, J.D. and Lewis, C. (1985)).
I present the data in simple, clean and clear charts (Meeks, E. (2015)) like bar charts, plot charts and pie charts (for data types with two categories). These chart types are chosen as they are well known and easily interpretable, whereas charts like violin and box plots may require learning and so a barrier to entry for UFC fans. The high contrast colours and black font on white background labels were visual encoding choices for maximum legibility and to make sure users can easily, accurately and correctly interpret the data (Healey, C.G. (2022)).

Despite my efforts and design choices some of the font sizes and text can be too small and congested in places which limits interpretation. An important improvement for V2.

Assumption 3:
The most watched UFC fights feature top contenders (those ranked highest in a weight class) (Davies, B. (2023)) so users will have the greatest interest in their data.

Inferred design choice: emphasizing top contenders for user engagement
Because of this I gave the option to filter by the top 5 or 10 contenders in a weight class (dynamically updating the data dashboard), so users could focus on what interests them most. I also gave an option to ‘Data Fight’ by comparing 2 fighters in a weight class because a UFC fight is 1 v 1, not an entire weight class, and direct comparison can create greater insight and clarity. I chose these features because they are novel and help to deliver the user message (Duarte, N. (2019)).

Conclusion:
The dashboard charts employ simple, clear designs with high contrast and generally legible labelling (Meirelles, I. (2013) ). However, issues like small font sizes and congested text in the age bar chart hamper readability and interpretation. Another issue is mixing data types like percentages and incremental numbers on charts that don’t start from zero may confuse users and reduce readability. This highlights improvements in labelling for enhanced user experience and engagement in Version 2.

Future versions could also expand their scope by adding filters for year and addressing sexist biases by including female fighters. Despite my belief in equality and inclusion in sports, time constraints and complexity led me to prioritize data filters relevant to the majority of UFC fans. I will address these issues if Version 2 is developed.
Overall, the dashboard provides valuable insights into UFC data, meeting my goal of providing UFC fans with a valuable tool for exploring fighter statistics.
